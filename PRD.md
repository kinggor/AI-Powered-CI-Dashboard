## 1. 项目背景与目标
### 1.1 背景
产品经理手动追踪多个竞争对手（官网、公众号、社交媒体、招聘信息等）耗时巨大，信息分散，难以形成结构化洞察，导致决策滞后。

### 1.2 目标
构建一个基于 MiMo 大模型和多 Agent 框架的自动化竞品看板，实现：
- 每日自动抓取指定竞品动态
- 自动生成结构化分析报告（含影响评估、行动建议）
- 支持历史趋势看板（GitHub README 作为展示）

## 2. 用户角色与场景
- **主要用户**：产品经理、市场分析师
- **使用场景**：每日早会前获取竞品摘要日报；每周战略会获取深度趋势分析

## 3. 技术架构
采用**多 Agent 协同架构**，基于 OpenCode/KiloCode 框架，调用 MiMo-V2.5-Pro 模型（原生 100 万上下文）。

**四大 Agent 角色**：
1. **总控 Agent（竞品分析专家）**：负责任务拆解、策略制定、结果整合。
2. **采集 Agent**：使用 Firecrawl / Tavily API 抓取指定 URL 列表，提取正文并去重。
3. **洞察 Agent**：对每篇内容进行摘要、多维分析（市场/产品/技术/营销）、影响评估。
4. **报告 Agent**：汇总每日洞察，生成日报/周报，自动更新 GitHub README 看板。

## 4. Token 消耗预估
- 每日监控 10 个竞品源，平均每源产生 5 条新内容（共 50 条）。
- 每条内容处理（抓取+摘要+分析）平均消耗 8k Token → 50×8k = 400k。
- 每日汇总报告消耗 50k Token。
- 每周一次深度趋势分析（利用历史 30 天数据，100 万上下文）消耗 1M Token。
- **预计月消耗约 12M + 4M = 16M Token**（保守估计）。
